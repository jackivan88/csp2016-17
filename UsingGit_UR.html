Git کی کا استعمال کرتے ہوئے

Git کی ورژن کنٹرول کے نظام کی ایک قسم ہے. ہم نے ان پروگراموں، دستاویزات، اور معلومات کے دیگر اقسام کے تبدیلیوں کا مشاہدہ ہماری مدد کے لئے ورژن کنٹرول کے نظام کا استعمال کرتے ہیں. یہ ہمارے کام میں غلطیوں کو توڑنا اور ہم اس سے رجوع کرنے کی ضرورت ہے تو ہماری معلومات کے تاریخ کو محفوظ کرنے کی اجازت دیتا ہے .

ہم مناسب طریقے سے کمانڈ لائن سے Git میں استعمال کرنے کے بارے میں جاننے کے لئے جا رہے ہیں. اپنے Git شیٹ کو دھوکہ دے باہر نکل جاؤ اور تم ان سب کو وقت استعمال کریں گے کے بعد مندرجہ ذیل کمانڈز کو اجاگر: git pull, git add, git commit, اور git push.

چوکی 2: مرتب Git میں

آپ پہلے ہی اس طبقے کے لئے Git ترتیب دیا ہے، اگرچہ، آگے بڑھو اور آپ کی ترتیبات دوبارہ چیک.

قسم git config --global user.name "Your Name", آپ کی اصل کا پہلا اور آخری نام کے ساتھ کی جگہ "آپ کا نام". پھر دبائیں درج کریں. یہ دوسرے ڈویلپرز جو Git کی مخزن میں تبدیلیاں جانتے مدد ملتی ہے.

قسم git config --global user.email "yourname@seoulchristian.org", آپ کے اسکول کا ای میل ایڈریس کے ساتھ "yourname@seoulchristian.org" کی جگہ. پھر دبائیں درج کریں. یہ دوسرے ڈویلپرز کو آپ سے رابطہ کرنے کے لئے جہاں معلوم ہے میں مدد ملتی ہے.

قسم git config --global color.ui auto اور پریس درج کریں. یہ مدد کرنے کے لئے کمانڈ لائن پیداوار میں کچھ رنگ کا اضافہ کر دیتی آپ کو زیادہ آسانی سے اس کو پڑھنے.

پھر ٹائپ git config --list اور پریس کو مناسب طریقے سے یقین ہے کہ آپ کو قائم کیا ہے ہر چیز کو بنانے کے لئے درج کریں.

چوکی 4: ذخیرہ بنائیں

اب ہم اس حقیقی کے لئے کوشش کرنے کے لئے جا رہے ہیں. کھولیں Git میں ایک شاندار ہڑتال (یا ٹرمینل میک پر) اور قسم mkdir projects اور پریس درج کریں. اس کے بعد استعمال کریں cd "منصوبوں" ڈائریکٹری اور قسم میں داخل کرنے کی git init usingGit اور پریس درج کریں. تم نے صرف آپ کے کمپیوٹر پر "usingGit" نامی ایک مخزن پیدا.

آپ "منصوبوں" ڈائریکٹری اور قسم میں اب بھی ہیں بات کو یقینی بنائیں git clone git@github.com:seoul-christian-school/csp2016-17.git اور پریس درج کریں .

چوکی 5: تبدیلیاں

Git میں ایک شاندار ہڑتال میں، آپ کے "usingGit" مخزن کے لئے جانا. قسم Git کی حیثیت اور پریس درج کریں. آپ کے پاس کوئی تبدیلیاں ہیں ایک پیغام تمہیں کہہ دیکھنا چاہئے. ابھی ٹائپ touch first.txt اور پریس درج کریں. پھر ٹائپ vim first.txt اور پریس درج کریں. اب آپ ایک مختلف انٹرفیس کو دیکھ لیں گے. یہ پر نصب تقریبا ہر یونیکس یا لینکس مشین طاقت ٹیکسٹ ایڈیٹر ہے. آپ دیکھ سکتے ہیں ایک اور ایک نینو ہے. فائل میں ترمیم کرنے کے لئے، آپ اپنے کی بورڈ پر خط میں دبانے سے "داخل کریں" کے انداز پر تبدیل کرنے کی ضرورت.

آپ ڈالیں موڈ میں ہیں اب کہ، قسم Here is some text. فائل میں. اس کے بعد معمول کے انداز پر واپس جانے کے لئے اپنے کی بورڈ پر فرار دبائیں. پھر ٹائپ :wq اور پریس کو بچانے کے لئے درج کریں (لکھنا) اپنی تبدیلیوں اور چھوڑ طاقت.

ابھی ٹائپ git status اور پریس درج کریں. آپ کو اس طرح کچھ نظر آنا چاہئے:

آپ یہ آپ کے اگلے کیا پر ہدایات دیتا ہے کہ محسوس کریں گے. آگے بڑھو اور ٹائپ git add first.txt اور پریس ورژن کنٹرول کرنے کے لئے فائل کو شامل کرنے کے لئے درج. اس کے بعد استعمال کریں git status ایک بار پھر تبدیلی ہوئی ہے اس کو دیکھنے کے لئے.

پھر ٹائپ git commit -m "Add first.txt to version control" اور پریس درج کریں. یہ Git کی میں فائل کے موجودہ ورژن کو ریکارڈ. -m پرچم Git میں بتاتا ہے کہ آپ کمانڈ کے بعد ایک پیغام ٹائپ کرنے جا رہے ہیں. آپ استعمال نہیں کرتے ہیں تو -m پرچم تو یہ آپ کو آپ کے پیغام ٹائپ کر سکتے ہیں، جہاں ایک طاقت ونڈو کو لے آئے گا.

آپ کے ہاتھ اٹھانے اور مسٹر Olinda اس چوکی کی توثیق کی ہے.

چوکی 6: شاخ

کبھی کبھی ہم مالک کے ورژن، جس سے Git میں "ماسٹر شاخ" کہا جاتا ہے کو پریشان کئے بغیر ایک منصوبے کے بعض حصوں پر کام کرنے کی ضرورت ہے. ایسا کرنے کے لئے، صرف ٹائپ git branch special اور پریس درج کریں. ابھی ٹائپ git branch آپ کے مخزن میں شاخوں کو دیکھنے کے لئے. آپ فی الحال پر ہیں میں سے ایک سبز ہے.

ابھی ٹائپ git checkout special اور پریس سپیشل برانچ کو سوئچ کرنے کے لئے درج. تبدیل کر دیا گیا ہے کو دیکھنے کے لئے ایک بار پھر Git کی شاخ کا استعمال.

اب، قسم vim second.txt اور پریس درج کریں. فائل میں کچھ متن کا اضافہ صرف کی طرح آپ کو آخری بار کیا اور اس کو بچانے. اس کے بعد استعمال کریں git add اور git commit -m Git میں میں نئی ​​فائل کو بچانے کے لئے. پھر ٹائپ git checkout master اور پریس ماسٹر شاخ پر واپس سوئچ کرنے کے لئے درج.

قسم git merge special اور پریس درج کریں. یہ آپ کے "مالک" شاخ "خصوصی" برانچ میں پیدا تمام معلومات ھیںچتی. آپ کو اس کو دیکھ، تو یہ ایک گلی کو واپس دو لین کے لئے ایک لین سے جاتا ہے، اور اس کے بعد کہ ایک سڑک کی طرح ہے. معلومات کے دو (یا اس سے زیادہ شاخیں) میں تقسیم کیا جاتا ہے اور پھر ایک میں واپس لے آئے.

آپ کے ہاتھ اٹھانے اور مسٹر Olinda اس چوکی کی توثیق کی ہے.

چوکی 7: حذف کریں اور کا نام تبدیل کریں

آپ نے پہلے ہی خارج کر دیں اور استعمال کرتے ہوئے فائلوں کا نام تبدیل کرنے کا طریقہ جانتے rm اور mv. آپ Git میں میں فائلوں کے ساتھ کام کر رہے ہیں، آپ سکنا ان آلات کا استعمال، لیکن اس کا استعمال کرنے کے لئے بہتر ہے git rm اور git mv کچھ اقدامات کو بچانے کے لئے.

ٹائپ کرنا touch deleteThisFile.txt renameThisFile.txt اور پریس درج کریں. پھر ٹائپ git add * ورژن کنٹرول کرنے کے لئے تمام ٹریک نہیں فائلوں کو شامل کرنے کے لئے. اس کے بعد استعمال کریں git commit -m اپنی تبدیلیوں کو محفوظ کرنے کے لئے.

ٹائپ کرنا git rm deleteThisFile.txt اور پریس درج کریں. یہ حکم دونوں ڈائریکٹری سے فائل کو ہٹا دیتا ہے اور بھی تبدیلی کے بارے میں Git میں بتاتا ہے. استعمال git status اس حکم کے اثرات کو دیکھنے کے لئے. آپ کو صرف فائل کو خارج کئے بغیر ورژن کنٹرول سے فائل کو خارج کرنا چاہتے تھے تو، آپ کو استعمال کریں گے git rm --cached.

استعمال کرنا git commit -m اپنی تبدیلیوں کو محفوظ کرنے کے لئے.

ٹائپ کرنا git mv renameThisFile.txt third.txt اور پریس درج کریں. یہ فائل نیا نام دیتا ہے اور تبدیلی کے بارے میں Git میں بتاتا ہے. استعمال git status اس حکم کے اثرات کو دیکھنے کے لئے.

استعمال کرنا git commit -m اپنی تبدیلیوں کو محفوظ کرنے کے لئے.

آپ کے ہاتھ اٹھانے اور مسٹر Olinda اس چوکی کی توثیق ہے.

چوکی 8: gitignore

Git میں کچھ فائلوں، فائلوں کی اقسام، یا ڈائریکٹریز کو نظر انداز کرنے کے لئے تشکیل کر سکتے ہیں. آپ ایک گروپ کے منصوبے پر کام کر رہے ہیں اور دوسرے صارفین کی مدد کے لئے نہیں جا رہے ہیں اس ڈائریکٹری میں نوٹوں رکھ رہے ہیں تو یہ مددگار ثابت ہو سکتا. تمہیں اس کی ضرورت، لیکن دوسرے لوگوں کو نہیں ہے، لہذا Git میں اس کو نظر انداز کرنے کے لئے آپ کو ترتیب.

سب سے پہلے، آپ کے "usingGit" ڈائریکٹری میں "ignore.txt" نامی ایک نئی فائل بنانے. پھر ٹائپ vim .gitignore اور پریس درج کریں. آپ طاقت میں ہیں ایک بار، پریس میں ترمیم کے موڈ میں داخل اور ٹائپ کرنا ignore.txt. پھر فرار اور قسم دبائیں :wq اور پریس کو بچانے اور چھوڑنے کی طاقت درج کریں. اب، آپ Git کی حیثیت کا استعمال کرتے ہیں تو، آپ کو صرف نیا .gitignore فائل میں شامل کیا جا کرنے کے لئے نظر آنا چاہئے. استعمالاور پریس کو بچانے اور چھوڑنے کی طاقت درج کریں. اب، آپ Git کی حیثیت کا استعمال کرتے ہیں تو، آپ کو صرف نیا .gitignore فائل میں شامل کیا جا کرنے کے لئے نظر آنا چاہئے. استعمالgit add اس فائل کو ٹریک اور پھر اس کا ارتکاب کرنے.

آپ کے ہاتھ اٹھانے اور مسٹر Olinda اس چوکی کی توثیق کی ہے.

چوکی 9: Stashing

آپ کو ایک مخزن میں تبدیلیاں کرتے ہیں، لیکن آپ ابھی تک ان کا ارتکاب نہیں کیا ہے، آپ تبدیلیاں محفوظ کریں اور انہیں مستقل طور پر حذف کیے بغیر کو تبدیل کر دیا ہے (یا اس سے بھی خارج کر دیا) فائلوں کر سکتے ہیں. یہ آپ کو ان کے بعد میں ان کو لاگو کرنے کو بچانے کے لئے اجازت دیتا ہے، ایک اور ارتکاب کے بعد بھی.

سب سے پہلے، کا استعمال کرتے ہوئے تین متن فائلوں کی تخلیق touch اور پھر Git میں انہیں شامل. اب، آپ کو استعمال کرتے ہیں تو git status آپ زیر التواء تبدیلیاں نظر آئیں گے. بلکہ اس کی بجائے استعمال کرتے ہوئے کی git commit, ٹائپ کرنا git stash save. تم جہاں فائلوں واقع ہیں کی وضاحت ایک پیغام نظر آئے گا. ابھی کا استعمال ls توثیق کرنے کے لئے نئی فائلوں موجود نہیں ہیں. ڈائرکٹری اپنے آخری ارتکاب کے بعد یہ کیا تھا کی طرح نظر آنا چاہئے. قسم git stash pop فائلوں کو واپس لانے کی. استعمال ls یہ یقینی بنانے کیلئے یہ کام کیا.

اس کے بعد، نئی فائلوں کو شامل کرنے اور ارتکاب. ابھی کا استعمال git rm ایک کمانڈ میں تمام تین فائلوں کو خارج کرنے. لیکن تم، استعمال کے ارتکاب سے پہلے git stash یاد میں تبدیلیوں کو محفوظ اور ڈائریکٹری کو بحال کرنے کے لئے (آپ کو آخر میں ٹائپ کرنا "بچانے" کی ضرورت نہیں) آپ کی آخری کرنے کا عہد. اب، آپ کی جگہ پر واپس فائلوں کو دیکھنا چاہئے. کا استعمال کرتے ہوئے git stash pop انہیں دوبارہ حذف کردے گا. آگے بڑھو اور شامل کریں اور تبدیلیوں کا ارتکاب.

آپ کے ہاتھ اٹھانے اور مسٹر Olinda اس چوکی کی توثیق کی ہے.

Cچوکی 10: ہسٹری

آپ کو دوسرے لوگوں کے ساتھ کام کر رہے ہیں خاص طور پر جب، یہ جب بعض تبدیلیاں کی گئی ہیں اور جو ان کو بنایا باہر تلاش کرنے کے لئے اہم ہے. آپ استعمال کرتے ہیں جب یہ ہے git log.

قسم git log اور کیا ہوتا ہے کو دیکھنے کے لئے دبائیں درج کریں. پیداوار بھی کم میں "پائپ کے ذ ریعے" ہے، لہذا آپ کو چھوڑ کرنے کے لئے اسپیس بار اور پریس (ق) کے ساتھ اس کے ذریعے سکرال کر سکتے ہیں. تمام ارتکاب کورس کی، آپ کی طرف سے ہونا چاہئے.

چوکی 13: گڈ Git کی مرتکب

اچھا Git کی مرتکب لئے سات قواعد ہیں. کرس بیم کی طرف سے سات قواعد کی اس شاندار وضاحت پڑھیں.. بنیادی طور پر، آپ کو آپ کے ارتکاب ہے کہ اس بات کو یقینی بنانے کی ضرورت ہے:

  1. ایک خالی لائن کے ساتھ جسم سے علیحدہ کریں مشروط
  2. 50 حروف پر عنوان کی سطر کی حد.
  3. عنوان کی سطر بڑا لکھیں
  4. ایک مدت کے ساتھ موضوع کی لائن ختم نہیں کریں
  5. موضوع کی لائن میں ضروری موڈ کو استعمال کریں
  6. 72 حروف جسم کو لپیٹ
